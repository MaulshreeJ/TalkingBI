============================================================
ARCHITECTURE DECISION: DATAFRAME REGISTRY
============================================================

Problem: LangGraph state is a TypedDict that must be
JSON-serializable between nodes. A pandas DataFrame cannot
live inside LangGraph state.

Solution: A sidecar module-level registry keyed by run_id:

  api/run.py       ->  register_df(run_id, df)    # before graph.invoke()
  graph/nodes.py   ->  get_df(run_id)             # inside query_node only
  api/run.py       ->  deregister_df(run_id)      # always cleaned up (finally)

============================================================
NODE EXECUTION LOGIC
============================================================

query_node:
  - Retrieves DataFrame from registry by run_id
  - Executes each KPI independently (one failure never blocks others)
  - Supported: sum, avg/mean, min, max, count, nunique
  - Supports groupby on time_column or segment_by
  - No eval() or exec() - pure pandas only

prep_node:
  - pd.DataFrame -> {"type": "timeseries", "data": [records]}
  - int/float    -> {"type": "scalar", "value": N}
  - Unknown      -> {"type": "unknown", "raw": str}  (never crashes)

insight_node:
  - Rule-based only (no LLM in Phase 2)
  - Timeseries: min/max value range across periods
  - Scalars: formatted number with comma separators

chart_node:
  - timeseries -> line chart spec with x_key/y_key auto-detected
  - scalar     -> metric card spec
  - Unknown    -> placeholder (never dropped)
