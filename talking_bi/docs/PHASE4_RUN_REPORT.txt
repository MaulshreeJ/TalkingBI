Date: 30 March 2026

============================================================
EXECUTIVE SUMMARY
============================================================

Phase 4 introduces Analytical Intelligence + LLM Layer to 
TalkingBI. The system has evolved from simply pulling math facts 
(descriptive) to genuinely understanding the data shape, 
recognizing trends, identifying missing data, and generating 
human-readable business narratives. 

CRITICALLY: We built this while keeping the LLM entirely 
sandboxed. The LLM never touches the raw data and never does 
math. It simply translates deterministic, python-calculated 
insights into conversational business summaries.

============================================================
ARCHITECTURAL DECISIONS
============================================================

1. FIXING DATA SEMANTICS (DeepPrep v2)
   In Phase 3, we replaced 'NaN' values with '0'. In Phase 4, we recognized that missing data is a critical business insight. DeepPrep now tracks `missing_count` and passes it forward, leaving the business logic intact.

2. STRUCTURED MULTI-TYPE INSIGHTS
   Insights are no longer simple strings. They are now strict dictionaries:
   {
       "kpi": "Total Revenue",
       "type": "trend | range | data_quality | scalar",
       "details": {...},
       "confidence": 0.0 - 1.0
   }

3. CONFIDENCE SCORING
   The system now scores its own insights. If a KPI is calculated on a column that is missing 20% of its data, the system flags the insight with a `0.80` confidence score.

4. LLM AS AN ENHANCEMENT, NOT A DEPENDENCY
   The `InsightNarrator` is the ONLY place the LLM is involved. 
   - It is fed strict rule-based facts.
   - It is explicitly instructed: DO NOT INVENT NUMBERS.
   - If the LLM goes down, the pipeline still returns the structured insights to the frontend perfectly.
